# Development Workflow

## Step-by-step process for each task

For every task in `tasks.md`, follow these steps:

1. Checkout from the latest `main` branch
2. Create a new branch for the task
3. Implement the task
4. Commit your changes
5. Push the branch to GitHub
6. Create a Pull Request into `main`

Only start the next task after the previous PR has been merged.
